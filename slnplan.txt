3.3 Default permission presets
When creating a user, admin picks a preset first, then fine-tunes:
Owner preset (all true) — full access.
Worker preset:
* ✓ viewCustomers, createCustomer, editCustomer
* ✓ generateBill, recordPayment
* ✓ viewComplaints, createComplaint, updateComplaintStatus
* ✓ logOwnHours, viewOwnSalary, viewOwnCollections
* ✗ everything else
Custom: admin ticks any combination.

3.5 Permission enforcement
Three layers of enforcement:

2. Menu level — sidebar items hidden if user lacks permission
3. Component level — buttons/actions conditionally rendered via <RequirePermission perm="editBill">
All three checked — direct URL access to restricted page redirects to first allowed page.

🔥 HIGH IMPACT (strongly recommend)
1. Data sync across devices — actually solve it
You said "I'll take care" but the core pain you mentioned was "saved on laptop doesn't show on phone." localStorage can't solve this. Options:
* Firebase Firestore — free tier, real-time sync across devices, 30 min setup

* Google Sheets as backend — zero-cost, works offline, export built-in, familiar to non-tech users

* "Install app" icon on phone home screen (looks like native app)
* Works in areas with weak signal
* Auto-sync when back online (if backend connected)
